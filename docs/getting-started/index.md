# Getting Started

This section walks you from zero to a running AO project with autonomous AI agents executing workflows continuously.

AO is designed for **autonomous execution**: you define work and the daemon runs it automatically, 24/7. Single synchronous runs (`--sync`) are useful for debugging, not the primary use case.

## Pages

- **[Installation](installation.md)** -- Use the upstream installer from `launchapp-dev/ao`, a release archive, or a source build.
- **[Quick Start](quick-start.md)** -- Initialize a repo, create a task, start the daemon, and watch workflows run autonomously.
- **[Project Setup](project-setup.md)** -- Understand the split between project-local `.ao/` config and repo-scoped runtime state.
- **[A Typical Day](typical-day.md)** -- End-to-end walkthrough of creating requirements, materializing tasks, and letting the daemon execute automatically.
