High-Risk Confirmation Dialog (Desktop, Invalid -> Valid Intent)

Stop daemon

High risk action

Stopping the daemon interrupts active workflow execution and can leave queued work waiting for manual resume.

Planned request

Action key
daemon.stop
Method + path
POST /api/v1/daemon/stop

Planned effects

  • Transition daemon lifecycle from `running` to `stopping`.
  • Pause execution dispatch for active runs.
  • Keep event history and diagnostics available for triage.

Preconditions

  • Current health must be `ok` before submit is allowed.
  • In-flight guard must be empty (`pendingAction = none`).
  • Success copy appears only after response revalidation checks pass.

Irreversible consequences

  • In-flight tasks may require manual restart.
  • Automation latency increases until daemon restart.

Rollback guidance: run `daemon.start` and validate `/api/v1/daemon/health`.

Phrase must match exactly, including letter case.

Valid input example: STOP DAEMON -> confirm becomes enabled and dispatch locks to a single pending request.
